## 任务清单
- 修改时间0127-184500、模型切换到Veo：根据用户提供的Python代码示例，将测试模型列表更新为优先测试Google Veo视频生成模型（veo-3.1-fast-generate-preview、veo-3.1-generate-preview、veo-2、imagen-3.0-generate-001），这些是真正的视频生成专用模型。同时在后端添加Veo模型检测逻辑和更详细的错误日志，以便诊断JavaScript SDK是否支持Veo的视频生成API。

- 修改时间0127-182000、错误提示优化：当视频生成失败时，不再只在控制台输出，而是通过弹框依次显示每个模型的尝试结果，包括成功/失败状态和详细错误信息，方便用户了解问题原因。同时提供诊断工具（test-connection.js）用于排查网络连接和API可用性问题。
- 修改时间0127-181500、模型优化：将视频生成接口（text2video）使用的模型更换为 `gemini-3-nano-banana-video-preview`，并实现智能回退机制。系统会按优先级尝试多个模型（gemini-3-nano-banana-video-preview → gemini-2.0-flash-exp → gemini-1.5-flash → gemini-1.5-pro → gemini-pro），确保在某个模型不可用时自动切换到下一个可用模型，提高视频生成的成功率和稳定性。
- 修改时间0127-180000、模型切换：将视频生成接口（text2video）使用的模型从 `gemini-3-pro-image-preview` 更换为 `gemini-3-pro`，因为前者主要用于图像生成，无法生成真实视频内容。同时优化了 Prompt 结构，更明确地要求输出视频格式（MP4/GIF），提高视频生成成功率。
- 修改时间0126-132000、配置优化：text2video Proxy 深度优化，为 `gemini-3-pro-image-preview` 模型增加 `generationConfig` 配置（temperature: 0.8）及更严格的 Prompt 约束（明确要求 Video/GIF 格式，拒绝静态图片），尝试解决模型倾向于生成静态 JPEG 的问题。
- 修改时间0126-131000、问题排查：针对 `text2video` 接口返回图片而非视频的问题，增强了 Prompt 指令（显式请求 Video/GIF），并在后台增加了 MIME 类型日志记录，以确认 `gemini-3-pro-image-preview` 模型的实际输出格式。
- 修改时间0126-130000、Bug修复：修复 text2video 接口在 Gemini 模型返回图片数据（如 image/jpeg）时仍标记为 video 类型导致前端播放器无法显示的问题。现在会根据返回数据的 MIME 类型自动判断并返回正确的 type（image 或 video），确保前端正确渲染结果。
- 修改时间0126-125000、功能增强：text2video Proxy 适配 Gemini 视频/图像生成，支持 `gemini-3-pro-image-preview` 模型（映射自“nano banana”），增加对 Base64 视频/图像数据的解析逻辑，确保生成的视频内容可以正确提取并播放。
